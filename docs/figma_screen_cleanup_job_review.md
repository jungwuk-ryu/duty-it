# Figma 화면정리 채용정보 반영 기록

작성일: 2026-06-19

## 확인한 디자인

- Computer Use로 현재 열려 있는 Figma 파일 `듀잇`의 `화면정리` 섹션을 확인했다.
- `채용정보` 프레임은 360x1034 모바일 화면으로, 다음 구성을 기준으로 한다.
  - 상단 로고, 알림 아이콘, 메뉴 아이콘
  - 검색 입력
  - `채용공고` 섹션 제목과 이동 화살표
  - `전체`, `필터`, `최신순` 컨트롤
  - 채용공고 리스트
  - `기업정보` 섹션과 2열 회사 카드
  - 하단 네비게이션 4탭: 행사, 채용, 북마크, 캘린더

## 발견한 차이

- 실제 앱은 채용 화면 안에 `채용/북마크` 상단 탭을 두고 있어 Figma의 `채용공고` 섹션 구조와 달랐다.
- 하단 네비게이션에 `북마크` 탭이 없었다.
- 로그아웃 상태의 채용 화면에서 알림 아이콘이 숨겨져 Figma와 달랐다.
- 채용 리스트 제목 타이포가 Figma보다 크게 보여 화면 밀도가 낮았다.
- `기업정보` 카드 영역은 API가 없어 실제 회사 데이터로 구현할 수 없는 상태였다.
- 채용 필터 빈 상태 문구가 `행사`로 되어 있었다.

## 반영한 수정

- 채용 화면 헤더를 `채용공고` 섹션 구조로 정리했다.
- 하단 네비게이션은 요청에 따라 `북마크` 탭을 숨기고 3탭 구조를 유지했다.
- 채용 화면 내부에서는 행사 화면과 같은 탭 UI로 `채용공고`와 `북마크`를 전환할 수 있도록 했다.
- 채용 화면에서는 로그인 여부와 관계없이 알림 아이콘을 보여주고, 로그아웃 상태에서는 로그인 화면으로 이동하도록 했다.
- 채용 리스트 타이포와 간격을 Figma 화면 밀도에 가깝게 조정했다.
- `기업정보`는 별도 API가 생길 때까지 숨김 처리했다.
- 채용 상세의 지도 프리뷰는 좌표 API가 준비될 때까지 숨김 처리했다.
- 채용 필터 빈 상태 문구를 `채용공고` 기준으로 수정했다.

## 검증

- `flutter analyze`를 변경 파일 범위로 실행했고 새 이슈는 없었다.
- XcodeBuildMCP로 iOS Simulator에 재빌드/재실행했다.
- Simulator에서 채용 탭을 열어 `채용공고`, 필터, 리스트, 하단 네비게이션이 표시되는 것을 확인했다.
- Simulator에서 하단 네비게이션이 `행사`, `채용`, `캘린더` 3탭으로 표시되는 것을 확인했다.
- 서브 에이전트 코드 리뷰를 받았고, 다음 사항을 추가 반영했다.
  - 북마크 목록 진입 후 하단 행사/채용 탭을 다시 누르면 원래 목록으로 복원되도록 수정
  - 알림 아이콘 탭 시점에 로그인 상태를 다시 판단하도록 수정
  - 기업정보 카드는 API 부재로 숨김 처리
  - 채용 상세 지도 프리뷰는 API 좌표 부재로 숨김 처리

## 남은 참고 사항

- `기업정보`는 별도 회사 API가 추가되면 실제 응답 기반으로 다시 노출해야 한다.
- 지도는 주소 텍스트만 있는 현재 API 상태에서는 숨김이며, 좌표 또는 안정적인 지오코딩 정책이 정해지면 다시 노출해야 한다.
- 전체 `flutter analyze`에는 기존 생성 파일/앱 진입점 관련 경고가 남아 있으며, 이번 수정 범위의 신규 문제는 아니다.

## 2026-06-20 북마크 화면 재구성 기록

- 행사/채용 화면 내부의 `행사/북마크`, `채용공고/북마크` 전환 탭은 제거했다.
- 하단 네비게이션에 `북마크` 탭을 다시 노출하고, `행사`, `채용`, `북마크`, `캘린더` 4탭 구조로 복원했다.
- 북마크 화면은 Figma의 `채용정보_북마크` 화면을 기준으로 구성했다.
  - 상단 로고/알림/메뉴
  - 검색 입력
  - 화면 내부의 `행사`, `채용` 전환 탭
  - `전체`, `필터`, `최신순` 컨트롤
  - 북마크한 행사/채용 목록
- 북마크 조회는 기존 API의 `bookmarked=true` 파라미터를 사용한다.
  - 행사: `/v2/events`
  - 채용: `/v1/job-postings`
- 북마크 해제 시 현재 북마크 목록에서 해당 항목이 즉시 제거되도록 별도 북마크 카드 동작을 구현했다.
- 로그인하지 않은 상태에서 하단 `북마크` 탭을 누르면 로그인 화면으로 이동하도록 유지했다.

## 2026-06-20 설정 화면 신고번호 정렬 기록

- `알림 설정` 행은 현재 별도 내부 알림 설정 화면이 없어 기기 앱 설정으로 이동하도록 연결했다.

  - `개인 정보 처리 방침`은 공개 URL이 확인되지 않아 임의 연결하지 않고 Figma와 같은 비동작 텍스트로 유지했다.

## 2026-06-20 알림 설정 화면 구현 기록

- Figma `알림 설정` 화면을 기준으로 설정 화면의 `알림 설정` 행을 내부 페이지로 연결했다.
- Swagger 기준 서버가 지원하는 알림 설정/구독만 구현했다.
  - 사용자 설정: `push`, `bookmark`, `calendar`, `marketing`
  - 구독 타입: `EVENT_KEYWORD`, `EVENT_HOST`, `EVENT_TYPE`, `JOB_KEYWORD`, `JOB_COMPANY`
  - 알림 조회: `/api/v2/alarms`는 행사 알림과 채용 알림을 함께 반환
- 채용 북마크 알림은 서버 설정/구독 타입에 별도 지원이 없어 비활성 안내로 표시했다.
- `calendar`, `marketing` 알림 설정은 서버가 지원하므로 새 알림 설정 화면의 푸시 알림 아래에 별도 토글로 노출했다.
- 맞춤 알림 관리 화면을 추가했다.
  - 행사: 키워드, 행사 유형, 주최 조건
  - 채용: 키워드, 북마크한 회사 조건
  - 서버 저장 전까지 필터 화면 내 선택은 로컬 상태로만 유지하고, `필터 적용` 시 구독 생성/삭제를 동기화
- 알림 목록은 V2 응답의 `target.event`, `target.jobPosting` 구조를 읽도록 보강했고, 채용 알림 탭 시 채용 상세로 이동하도록 했다.
- 검증:
  - `dart analyze` 변경 범위 통과
  - `flutter test test/app/modules/settings/settings_view_controller_test.dart` 통과
  - `git diff --check` 통과
  - `flutter analyze`는 기존 generated/model/main 경고 6건으로 실패하며, 이번 변경 범위 신규 이슈는 없음
  - XcodeBuildMCP로 iOS Simulator 빌드/실행 성공
  - Computer Use로 앱 설정 -> 알림 설정 -> 채용/행사 맞춤 알림 필터 진입을 확인
  - 실제 사용자 구독 데이터 변경을 피하기 위해 `필터 적용`은 누르지 않음
- 서브 에이전트 리뷰 반영:
  - 맞춤 알림 저장 시 신규 조건 생성을 먼저 수행하고, 성공 후 기존 조건 삭제를 수행하도록 순서를 변경했다.
  - 알림 설정 토글 업데이트는 잠금 안에서 최신 사용자 설정을 다시 읽도록 변경했다.
  - 비로그인 직접 접근 시 구독 API를 호출하지 않도록 방어 로직을 추가했다.
  - Swagger에서 `/api/v1/subscriptions`가 배열 응답이고 회사 응답 회사명이 `corpNm`임을 재확인했다.

- 서브 에이전트 리뷰를 반영해 모달 컨트롤러에 인스턴스 태그를 붙이고, 알림 필터에서 열 때 기존처럼 주최 목록을 `NAME ASC`로 요청하도록 했다.
- 검증:

  - `flutter test test/app/modules/settings/settings_view_controller_test.dart` 통과

## 2026-06-20 필터 화면 앱바 Divider 정리 기록

- 행사/채용공고 맞춤 알림 필터 화면의 앱바와 콘텐츠 사이 구분선을 공용 `SectionDivider` 사용 흐름으로 정리했다.
- 알림 필터 화면은 `SimpleAppBar`가 이미 포함하는 `SectionDivider`만 사용하도록 중복 divider를 제거했다.
- 채용 검색 필터 화면의 직접 `ColoredBox` divider 구현은 `SectionDivider`로 교체했다.
- 검증:
  - 변경 파일 범위 `dart analyze` 통과
  - 직접 구현된 동일 divider 패턴이 더 이상 남아 있지 않음을 `rg`로 확인

## 2026-06-20 채용 북마크 알림 숨김 기록

- 서버에서 북마크한 채용공고 알림을 별도 설정/구독 타입으로 지원하지 않으므로 알림 설정 화면에서 해당 행을 숨김 처리했다.
- 향후 서버 지원 시 재노출할 수 있도록 `_UnsupportedToggleRow` 구현과 화면 내 조건부 코드 경로는 남겨두었다.
- 채용 섹션에는 서버가 지원하는 `채용공고 맞춤 알림`과 관리 진입만 노출한다.

## 2026-06-20 알림 설정 카테고리 Divider 복원 기록

- Figma 알림 설정 화면 기준으로 `행사`, `채용` 카테고리 제목 아래의 얇은 divider를 복원했다.
- 기존 앱바/콘텐츠 구분용 `SectionDivider`는 8px 밴드라 카테고리 내부에는 사용하지 않고, 화면 내부에 1px `Divider`를 적용했다.
- 채용 북마크 알림 숨김 상태에서도 `채용공고 맞춤 알림` 행 위 간격이 어색하지 않도록 카테고리 헤더와 divider를 한 묶음으로 정리했다.

## 2026-06-20 새 알림 페이지 구현 기록

- Figma `알림` 새 화면(`node-id=3008-42562`) 기준으로 알림 목록 페이지를 재구성했다.
- 서버 소스 `klaus9267/duit_server` 기준으로 `/api/v2/alarms`가 이벤트와 채용공고 알림을 함께 반환하며, `target.targetType`/`target.event`/`target.jobPosting` 구조를 지원함을 확인했다.
- 서버의 `AlarmPaginationParam`에는 `targetType` 필터가 없으므로 `전체`, `행사`, `채용` 탭은 클라이언트에서 `event`/`jobPosting` 및 알림 타입 문자열로 분류한다.
- 상단 설정 아이콘은 기존 알림 설정 화면으로 이동한다.
- 행사/채용 탭에서 맞춤 알림 조건이 없으면 Figma의 조건 안내 배너를 표시하고, 배너 탭 시 해당 맞춤 알림 조건 화면으로 이동한다.
- 알림 아이템은 Figma처럼 원형 로고 없이 텍스트 중심 목록으로 변경하고, 읽지 않은 알림은 연한 붉은 배경으로 강조한다.
- 검증:
  - 변경 파일 범위 `dart analyze` 통과
  - `flutter run -d DB647BCF-590A-455D-8834-BD0A97365E27 --debug --no-resident` 성공
  - Simulator에서 알림 페이지 진입, 전체/행사/채용 탭 전환, 조건 배너, 배너의 맞춤 알림 조건 화면 이동, 설정 아이콘 이동 확인
- 서브 에이전트 리뷰 반영:
  - 서버가 탭별 알림 필터를 지원하지 않는 제약 때문에, 선택 탭의 매칭 알림이 뒤 페이지에 있을 수 있는 문제를 보완했다.
  - 선택 탭에 매칭 알림이 없고 다음 페이지가 남아 있으면 빈 상태를 바로 보여주지 않고 추가 페이지를 확인한 뒤 확정한다.
  - 페이지 끝 이후 추가 요청을 막고, 알림 API 실패 시 빈 상태 대신 재시도 UI를 표시한다.
  - 설정 화면 복귀 후 구독 조건을 무조건 다시 불러오도록 수정했다.
  - `ListView(children: ...)` 대신 `CustomScrollView`와 `SliverList.builder`로 바꿔 로드된 알림 행을 lazy build한다.

## 2026-06-23 채용 지역 필터 원인 확인 및 우회 기록

- 라이브 API에서 `workRegions=SEOUL` 또는 `workRegions=GYEONGBUK`을 보내면 HTTP 200이지만 `content`가 빈 배열로 내려오는 것을 확인했다.
- 같은 최신 채용공고 API를 지역 조건 없이 호출하면 서울, 경북 등 지역 공고가 실제로 포함되어 있어 데이터 자체가 없는 문제는 아니었다.
- 서버 소스 `duit_server`의 `JobPostingRepositoryImpl`은 `jobPosting.workRegion.startsWith(it.displayName)`으로 필터링한다.
- 라이브 응답의 `workRegion`은 `(07516) 서울특별시...`, `(37560) 경상북도...`처럼 우편번호와 장문 주소로 시작하므로 서버의 `startsWith("서울")`, `startsWith("경북")` 조건과 맞지 않는다.
- 클라이언트는 서버 지역 필터가 고쳐질 때까지 `workRegions` 파라미터를 보내지 않고, 내려받은 `workRegion`/`location`/`workDistrict`/`companyAddress`를 파싱해 로컬에서 지역 필터를 적용하도록 변경했다.
- 채용 목록과 북마크한 채용공고 목록 모두 같은 `JobFilterMatcher`를 사용해 지역, 경력, 종료 공고 필터 판정을 공유한다.
- 검증:
  - 변경 파일 범위 `dart analyze` 통과
  - `flutter test test/app/services/job_filter_matcher_test.dart test/app/services/job_filter_service_test.dart test/app/core/models/job_posting_test.dart` 통과
  - Simulator에서 채용 필터 지역 `서울` 적용 후 서울 채용공고가 빈 목록 없이 표시되는 것을 확인
  - 전체 `flutter analyze`는 기존 generated/model/main 경고 6건으로 실패하며, 이번 변경 범위 신규 이슈는 없음
- 서브 에이전트 리뷰:
  - 정확히 한 명에게 리뷰를 요청했으나 Codex 사용량 제한으로 서브 에이전트가 완료되지 못했다.
  - 사용자 지시에 따라 진행했고, 로컬 분석/테스트/API 확인/Simulator QA로 보완 검증했다.
